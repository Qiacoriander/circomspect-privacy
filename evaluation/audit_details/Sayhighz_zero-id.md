# 隐私泄露审计报告：Sayhighz_zero-id (泰国公民身份证验证)

## 1. 项目与漏洞概述
- **项目名**: `Sayhighz_zero-id`（零知识身份系统——泰国居民国民身份证）
- **泄露类型**: **1 Full Leak (完全侧漏)**
- **涉及信号**: `idDigits[13]` (泰国公民身份证 13 位完整号码)

---

## 2. 漏洞发生链路分析

### 审计目标电路：`Sayhighz_zero-id\circuits\thai_citizen.circom`

这是一个令人印象深刻的**零知识泰国公民身份证验证**电路，实现了完整的泰国 13 位身份证号校验算法：
1. 每位数字范围验证（0-9）：`check[i] <== idDigits[i] * (9 - idDigits[i] + 1)`
2. 加权校验和：`sum[i] = sum[i-1] + idDigits[i] * (13-i)`（权重从 13 递减到 2）
3. Mod 11 取余 + Mod 10 校验位计算
4. 最后一位与校验位比较

代码甚至用泰语写了注释（"เลขบัตร 13 หลัก" = "13位身份证号"）。

**致命问题**：整个 13 位身份证号 `idDigits[13]` 被作为私有输入传入，但在校验过程中：
```circom
    sum[0] <== idDigits[0] * 13;
    for (var i = 1; i < 12; i++) {
        sum[i] <== sum[i-1] + idDigits[i] * (13 - i);
    }
```
每个 `idDigits[i]` 都参与了确定性的线性运算并通过 `isValid` 输出和约束系统间接暴露。引擎通过分析 `sum`、`checkDigit`、`diff`、`isValid` 等中间信号的约束链，判定所有 13 位数字可被联立求解。

## 3. 审计结论：确认为有效泄露 (True Positive)
这是一个**极具现实意义的隐私悲剧**——零知识身份验证的核心目标就是"证明你持有有效身份证而不泄露号码"，但此电路反而把**完整的 13 位身份证号**全部暴露了。引擎还额外发现了 `<--` 赋值运算符和 under-constrained signal 等安全隐患。**True Positive**。
