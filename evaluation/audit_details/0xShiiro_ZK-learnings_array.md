## 2. 漏洞发生链路分析与复盘

### 审计目标电路：`0xShiiro_ZK-learnings\array.circom`

```circom
template Powers(n) {
    signal input a;
    signal output powers[n];
    
    powers[0] <== a;
    for (var i = 1; i < n; i++) {
        powers[i] <==  powers[i - 1] * a;
    }
}
```

### 代数泄露复盘
此电路意图对单一的隐私输入信号 `a` 计算出它的多次方（比如 $a^1, a^2, a^3...$）形成序列。
这是一个没有任何悬念的裸奔代码，它在首行就直接安排了 `powers[0] <== a`，将本应隐私的 `a` 直接挂入输出位，引发引擎一级警报。但即使我们抛开这最愚蠢的第一行：
如果开发者将第一行改为输入隐藏，仅仅利用后续的乘积操作（即仅输出 $a^2, a^3...$ 之流）：如果底数是单变量，且幂函数的乘法序列没有其他盲化因子的保护，只要 $a$ 未经随机标量掩盖或杂凑脱敏，在有理数和有限域下皆可被枚举甚至代数逆推攻破。
